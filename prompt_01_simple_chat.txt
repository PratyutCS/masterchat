6.  Database: MongoDB only.
7.  Frontend: Android Native (Java, XML, MVVM).
8.  Code must be clean, readable, and well-commented.
9.  Prefer clarity over cleverness.
10. Provide runnable code examples wherever helpful.

HALLUCINATION PREVENTION:

-   Do NOT invent libraries that do not exist.
-   Do NOT assume infrastructure that was not requested.
-   If a choice is uncertain, choose the simplest widely-used option.
-   Every API, event, and schema must be explicitly defined.
-   Do NOT skip steps with phrases like “implement as needed”.
-   Provide concrete file names and folder structures.
-   All examples must be realistic and executable.

FAST DEVELOPMENT PRINCIPLES:

-   Minimize moving parts.
-   Prefer monolithic backend.
-   Avoid premature optimization.
-   Use straightforward Socket.IO rooms.

-   Keep schemas minimal but extensible.

TARGET SCALE:

-   Personal use (≈ 5–50 concurrent users)
-   Focus on reliability and clarity, not hyperscale.

====================================================================

GOAL: Build a clean, minimal real-time 1-to-1 chat application from
scratch.

TECH STACK:

Backend: - Node.js (JavaScript) - Express - MongoDB with Mongoose -
Socket.IO - JWT auth - bcrypt

Android: - Java - MVVM - XML layouts - Retrofit - OkHttp - Socket.IO
Java client

FEATURES TO IMPLEMENT:

AUTH:

-   Register endpoint
-   Login endpoint
-   JWT access token
-   Password hashing with bcrypt
-   Auth middleware

USERS:

User schema fields:

-   _id
-   username (unique)
-   email (unique)
-   passwordHash
-   lastSeen
-   createdAt

CONVERSATIONS:

Private only.

Fields:

-   _id
-   members [userId]
-   createdAt

MESSAGES:

Fields:

-   _id
-   conversationId
-   senderId
-   text
-   status (sent/delivered/read)
-   createdAt

REAL-TIME FLOW:

When user opens chat:

1.  socket connects
2.  client emits authenticate with JWT
3.  server validates
4.  user joins conversation room
5.  messages flow via room

SOCKET EVENTS:

Client → Server:

-   authenticate
-   join_conversation
-   send_message
-   mark_read

Server → Client:

-   receive_message
-   message_delivered
-   message_read
-   user_online
-   user_offline

ANDROID SCREENS:

1.  Login
2.  Register
3.  Conversation list
4.  Chat screen

Provide example message bubble XML.

-   Folder structures
-   Complete Mongoose schemas
-   Express routes
-   Socket server setup
-   Android MVVM skeleton
-   Example Retrofit interface
-   Example socket manager
-   Step-by-step run instructions
