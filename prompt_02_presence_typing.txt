CONTEXT: Phase 1 chat is fully working.

GOAL: Add accurate presence and typing indicators WITHOUT Redis.

APPROACH:

Use in-memory tracking on the Node server.

PRESENCE DESIGN:

Maintain a Map:

userId -> set of socketIds

On connect:

-   mark online
-   broadcast user_online

On disconnect:

-   remove socket
-   if no sockets remain → broadcast offline
-   update lastSeen in Mongo

TYPING INDICATORS:

Events:

Client emits:

-   typing_start
-   typing_stop

Server broadcasts to room.

ANTI-SPAM:

Implement:

-   debounce on client (300–500ms)
-   auto-timeout on server (3–5 seconds)

ANDROID UPDATES:

Chat screen must show:

-   “typing…”
-   online dot
-   last seen text

Provide ViewModel logic example.

EDGE CASES:

-   multiple devices
-   quick reconnect
-   app backgrounded
-   socket reconnect

DELIVERABLES:

-   Updated socket handlers
-   Presence manager module
-   Typing manager module
-   Android UI updates
