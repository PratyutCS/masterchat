PHASE 6 — SIMPLE MUSIC CO-LISTENING

IMPORTANT GLOBAL RULES (APPLY TO ALL PHASES):

1.  Keep the system SIMPLE and beginner-friendly.
2.  DO NOT use Redis.
3.  DO NOT use Docker.
4.  DO NOT use Kubernetes or microservices.
5.  Use a single Node.js + Express server.
6.  Database: MongoDB only.
7.  Frontend: Android Native (Java, XML, MVVM).
8.  Code must be clean, readable, and well-commented.
9.  Prefer clarity over cleverness.
10. Provide runnable code examples wherever helpful.

HALLUCINATION PREVENTION:

-   Do NOT invent libraries that do not exist.
-   Do NOT assume infrastructure that was not requested.
-   If a choice is uncertain, choose the simplest widely-used option.
-   Every API, event, and schema must be explicitly defined.
-   Do NOT skip steps with phrases like “implement as needed”.
-   Provide concrete file names and folder structures.
-   All examples must be realistic and executable.

FAST DEVELOPMENT PRINCIPLES:

-   Minimize moving parts.
-   Prefer monolithic backend.
-   Avoid premature optimization.
-   Use straightforward Socket.IO rooms.
-   Use simple JWT auth.
-   Keep schemas minimal but extensible.

TARGET SCALE:

-   Personal use (≈ 5–50 concurrent users)
-   Focus on reliability and clarity, not hyperscale.

====================================================================

CONTEXT: Voice calling works.

GOAL: Implement simple synchronized music listening for small friend
group.

SIMPLIFICATION:

Host device is authoritative.

No complex drift correction required.

BASIC FLOW:

1.  Host plays track
2.  Host emits music_state
3.  Others align playback
4.  Periodic resync every 10s

MUSIC SESSION STATE:

Store in memory on server:

{ conversationId, trackUrl, positionMs, isPlaying, updatedAt }

SOCKET EVENTS:

-   music_play
-   music_pause
-   music_seek
-   music_state

ANDROID:

Use ExoPlayer.

Must implement:

-   join session
-   sync to host timestamp
-   basic latency adjustment

EDGE CASES:

-   late join
-   host leaves
-   pause/play spam

DELIVERABLES:

-   music state manager
-   socket handlers
-   Android ExoPlayer sync example
-   resync algorithm pseudocode
