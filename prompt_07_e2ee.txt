CONTEXT: Full chat app works.

GOAL: Add OPTIONAL end-to-end encryption for messages.

IMPORTANT:

Keep implementation understandable.

APPROACH:

Use:

-   AES-256-GCM for message encryption
-   RSA or Curve25519 for key exchange (choose simplest widely
    supported)

SIMPLIFIED MODEL:

Per conversation shared secret.

FLOW:

1.  Users exchange public keys
2.  Derive shared key
3.  Messages encrypted on sender device
4.  Server stores ciphertext only
5.  Receiver decrypts locally

MESSAGE FORMAT:

{ ciphertext, iv, authTag }

SERVER ROLE:

-   NEVER decrypt
-   treat payload as opaque

ANDROID:

Use:

javax.crypto

Provide example encryption helper.

KEY STORAGE:

Use Android Keystore.

EDGE CASES:

-   new device joins
-   key rotation
-   backward compatibility (unencrypted chats)

DELIVERABLES:

-   encryption architecture
-   key exchange flow
-   updated message schema
-   Android crypto helper
-   migration strategy
